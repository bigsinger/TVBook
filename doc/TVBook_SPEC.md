# TVBook Android TV 开发规范

## 1. 项目定位

TVBook 是面向 Android 电视盒子/电视机的离线绘本与音频播放器。目标使用方式是把内容和配置预先放入 U 盘根目录的 `tvbooks` 文件夹，电视端插入 U 盘后直接读取菜单、绘本、音频、收藏和播放记录。

应用必须优先适配遥控器操作，触控只作为兼容入口。界面元素需要可聚焦、可用方向键导航，并在焦点状态下有明显视觉反馈。

## 2. Android 兼容性

当前代码中的最低版本来自 `app/build.gradle`：

- `minSdkVersion 17`
- 对应 Android 4.2 Jelly Bean

后续开发以 Android 4.2 / API 17 作为最低支持版本。实现约束如下：

- 不引入需要 AndroidX/AppCompat 的控件依赖。
- 不使用 API 17 以上的未保护调用；确需使用高版本 API 时必须通过 `Build.VERSION.SDK_INT` 判断。
- 使用 Java 8 语法时以当前 Gradle 配置为准，避免引入额外运行时库。
- U 盘与外部存储访问继续保留传统文件路径扫描，同时兼容模拟器 `/data/local/tmp/tvbooks` 测试目录。
- 面向本地安装和电视盒子使用，优先保证 Android 4.x 到 Android 9 的文件路径访问稳定。

## 3. U 盘目录规范

所有内容、配置、播放记录和收藏都必须位于 U 盘根目录的 `tvbooks` 文件夹内。应用可以读取和创建以下结构：

```text
tvbooks/
  config/
    settings.properties
    playback.properties
    favorites.properties
    catalog.properties
  huiben/
    index.txt
    chitang/
      1.jpg
      1.mp3
      2.jpg
      2.mp3
  gushi/
    001.mp3
    002.mp3
```

目录约定：

- 一级至三级菜单直接来自文件夹层级，不依赖 `menu.txt`。
- `config/` 由应用自动创建，存放所有运行产生的数据。
- `catalog.properties` 保存一级至三级目录关系，不保存媒体文件名或数量。
- 绘本分类目录可以包含 `index.txt`，一行一个绘本目录名，用于固定播放顺序。
- 绘本页面使用同名序号文件，例如 `1.jpg` 和 `1.mp3`。页面可以只有图片或只有音频。
- 音频播放器进入可播放目录后仍按原有逻辑扫描媒体文件；主界面菜单不扫描媒体文件。
- 应用不得把收藏、播放进度、用户设置作为主存储写入应用私有目录。旧版本 SharedPreferences 只允许作为迁移或 U 盘不可写时的临时兼容。

## 4. 自动菜单规则

- `tvbooks` 的直接子目录是一级菜单。
- 一级目录的直接子目录是二级菜单，二级目录的直接子目录是三级菜单。
- 菜单最多展开三级；目录扫描只获取直接子目录，不递归统计文件。
- 到达叶子目录时，根据直接媒体文件快速识别绘本或音频，并使用原有播放器入口。

应用会额外提供“我的收藏”入口。

每个普通分类也必须在内容列表顶部提供一个虚拟“我的收藏”入口。该入口不对应真实媒体目录，只从 `favorites.properties` 中筛选当前分类路径下的收藏索引。

## 5. 目录缓存与同步

主界面必须优先保证遥控器上下左右响应流畅，不允许在焦点移动或菜单刷新时同步遍历大目录统计数量。

- 普通导航优先读取 `tvbooks/config/catalog.properties` 中已有的目录缓存。
- 缓存不存在时提示用户同步目录；菜单导航期间不扫描移动硬盘。
- 主界面提供“同步目录”按钮，并支持在主界面按 `Menu` 触发同步。
- 同步过程在后台线程执行，完成后写入 `catalog.properties` 并 Toast 提示。
- 同步内容只包括一级、二级、三级目录关系，不读取媒体文件或统计文件数量。
- 播放器进入具体绘本或音频目录时，可以为实际播放进行必要扫描，但不得影响主界面导航流畅度。
- 主界面右侧必须清晰标注“自动间隔(秒)”设置，避免用户无法理解加减按钮含义；该数值用于绘本自动翻页和音频连续播放的等待间隔。

## 6. 遥控器操作规范

通用按键：

- `方向键`：移动焦点或切换内容。
- `OK/Enter`：主菜单中打开焦点项；播放器中暂停/继续。
- `Back`：播放页控制栏可见时先隐藏控制栏；控制栏隐藏时先提示，再按一次退出播放器。
- `Menu`：主界面触发目录同步；播放器中显示/隐藏底部控制条。
- `Bookmark` / `*` / 红色功能键：收藏或取消收藏当前内容。

绘本播放器：

- `Left`：上一页。
- `Right`：下一页。
- `Up`：上一本。
- `Down`：下一本。
- `OK`：暂停或继续当前音频。
- `Menu`：显示/隐藏底部小控制条。
- `Bookmark` / `*` / 红色功能键：收藏/取消收藏当前绘本。

音频播放器：

- `Left`：快退当前音频。
- `Right`：快进当前音频。
- `Up`：上一首。
- `Down`：下一首。
- `OK`：暂停或继续。
- `Menu`：显示/隐藏控制条。
- `Bookmark` / `*` / 红色功能键：收藏/取消收藏当前音频。
- 普通播放状态按 `Left` / `Right` 快退/快进时，必须显示底部控制组和进度条，以便用户看到当前位置。

界面按钮也必须支持同等操作，便于部分遥控器通过焦点按钮完成控制。

控制栏交互：

- 播放页暂停或按 `Menu` 显示控制栏后，焦点必须进入底部控制按钮。
- 控制栏可见时，`Left` / `Right` 只在控制栏按钮之间移动焦点，不触发翻页、切歌或上下本。
- 音频控制栏的进度条必须可聚焦；进度条获得焦点时，`Left` / `Right` 调整播放进度，`Down` 回到按钮区。
- 音频只能在 MediaPlayer 实际完成回调后自动切换下一首，不使用绘本翻页间隔或完成看门狗。
- 控制栏可见时，`OK/Enter` 只执行当前焦点按钮。
- 控制栏可见时，`Back` 隐藏控制栏并把焦点交回播放画面。

## 7. 收藏规范

收藏数据文件：

```text
tvbooks/config/favorites.properties
```

收藏项最少保存：

- 类型：`audio_image` 或 `audio`
- 相对路径：相对于 `tvbooks/` 的内容路径
- 标题：界面显示名称
- 收藏时间：用于稳定生成“我的收藏”顺序播放列表；旧数据缺失时按标题和路径排序兼容。

行为要求：

- 绘本收藏到具体绘本目录，例如 `huiben/chitang`。
- 音频收藏到当前音频文件，例如 `gushi/100shou/001.mp3`。
- 再次按收藏键会取消收藏。
- 主菜单“我的收藏”根据收藏文件生成按钮列表。
- 每个普通分类内容列表顶部显示该分类范围内的“我的收藏”，进入后先显示该分类收藏明细。
- 打开全局“我的收藏”中的绘本时，播放器按全部收藏绘本形成虚拟播放列表，并定位到所选绘本。
- 打开全局“我的收藏”中的音频时，播放器按全部收藏音频形成虚拟播放列表，并定位到所选音频。
- 打开分类“我的收藏”中的具体收藏项时，播放器只按该分类路径下的收藏项形成虚拟播放列表，并定位到所选内容。
- 收藏播放列表只保存索引，不复制媒体文件；自动播放下一本/下一首时继续在该收藏列表内顺序播放。

## 8. 断点续播规范

播放记录文件：

```text
tvbooks/config/playback.properties
```

绘本记录：

- 当前分类最后播放的绘本序号。
- 当前绘本最后播放的页码。
- 当前页音频播放进度，单位毫秒。

音频记录：

- 当前音频目录最后播放的文件序号。
- 当前音频文件播放进度，单位毫秒。

保存时机：

- 进入新绘本、新页面、新音频时保存索引。
- 应用暂停、返回、退出播放器前保存当前媒体进度。
- 手动切换下一页/下一首时，新内容的媒体进度重置为 0。

恢复策略：

- 从普通菜单进入分类时，恢复该分类最近播放位置。
- 从收藏进入时，优先定位收藏内容，再恢复该内容的页码或媒体进度。
- 从收藏列表入口进入时，恢复该收藏列表最近播放位置；从具体收藏项进入时，优先定位该收藏项。
- 如果记录指向的文件已不存在，自动回退到目录中第一个可播放内容。

自动播放策略：

- 绘本页面自动翻页时间取“用户设置的自动间隔”和“当前页音频实际播放完成时间”二者中的较晚者。
- 当前页没有音频或音频短于自动间隔时，按自动间隔翻页。
- 当前页音频长于自动间隔时，等待音频完成后再翻页。
- 绘本或图集到最后一页后，满足自动翻页条件时进入下一本；收藏播放列表内继续按收藏列表顺序进入下一本。
- 手动翻页或切换绘本时，旧页面的定时器和旧音频完成事件必须失效，不能触发当前页重复播放或重复翻页。
- 每次页面音频播放必须有独立请求标识；旧 `MediaPlayer` 或旧音频完成回调不得影响当前页面。
- 不允许通过重播当前音频来处理疑似提前完成回调；应按已知音频时长设置完成兜底，确保不会重复播同一页音频，也不会因完成回调丢失而卡在当前页。

## 9. 界面规范

主界面：

- 左侧为一级菜单和全局设置按钮。
- 保持 1.5 固定三栏布局；二级和三级菜单共用中间区域，三级打开时替换二级列表，返回时恢复二级列表。
- 右侧为操作说明和状态信息。
- 按钮高度、字号、间距需要适合电视远距离观看。
- 焦点按钮需要明显高亮。

播放器界面：

- 绘本图片应铺满可用区域并居中显示，默认状态下优先保证大屏沉浸感。
- 绘本播放器默认隐藏按钮和信息条，不在左上角显示会遮挡画面的状态文字。
- 按 `Menu` 键后，在播放页底部浮出一排小控制按钮；控制条不得占据主要阅读区域，再按 `Menu` 隐藏。
- 绘本和音频底部控制栏显示后，连续 10 秒无操作必须自动隐藏；有操作时重新计时。
- 绘本控制条底部显示页码进度、上一页/下一页、上一本/下一本、暂停/继续、收藏和自动/手动。
- 音频播放画面不常驻左上角或中央信息文字；名称、序号、收藏状态与播放模式只在底部控制栏显示。
- 控制按钮使用遥控器可聚焦按钮，作为可见辅助入口；常用播放动作以遥控器方向键和 OK 键为主。

## 10. 实现原则

- 先保证离线 U 盘播放能力，再考虑在线 H5。
- 不破坏现有 `audio_image` 和 `audio` 菜单行为。
- 文件读写失败必须用 Toast 或状态文本提示，不应直接崩溃。
- 文件排序应稳定，数字文件名按自然数字顺序排列。
- 代码保持简单：优先使用 Java/Android 标准库，避免引入新依赖。
- 新增功能必须能在模拟器通过 `/data/local/tmp/tvbooks` 测试目录验证。

## 11. 验证清单

开发完成后至少验证：

- Gradle Debug 包可以成功构建。
- `minSdkVersion` 仍为 17。
- 模拟器存在 `/data/local/tmp/tvbooks` 目录树时可自动生成菜单。
- 主菜单可用方向键移动焦点并打开内容。
- 主菜单普通导航不触发目录同步扫描；按“同步目录”或 `Menu` 后会生成/更新 `catalog.properties`。
- 绘本可以上下本、左右翻页、OK 暂停、Menu 显示/隐藏控制条、Bookmark 收藏。
- 音频可以左右快退/快进、上下切歌、OK 暂停、Menu 显示/隐藏控制条、Bookmark 收藏。
- 音频快退/快进时底部进度条可见，进度条获得焦点后左右键继续调整进度。
- 绘本自动播放按自动间隔和当前页音频完成时间二者的最大值切页，手动翻页不会导致音频重复播放。
- 全局和分类“我的收藏”均可显示收藏明细，并进入虚拟收藏播放列表按收藏顺序自动播放。
- 退出后再次进入可以恢复绘本页码和音频进度。
- 收藏文件、播放记录、设置文件写入 `tvbooks/config/`。
- U 盘目录不存在或菜单缺失时有明确提示，不崩溃。
