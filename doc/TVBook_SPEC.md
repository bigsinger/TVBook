# TVBook Android TV 开发规范

## 1. 项目定位

TVBook 是面向 Android 电视盒子/电视机的离线绘本与音频播放器。目标使用方式是把内容和配置预先放入 U 盘根目录的 `tvbooks` 文件夹，电视端插入 U 盘后直接读取菜单、绘本、音频、收藏和播放记录。

应用必须优先适配遥控器操作，触控只作为兼容入口。界面元素需要可聚焦、可用方向键导航，并在焦点状态下有明显视觉反馈。

## 2. Android 兼容性

当前代码中的最低版本来自 `app/build.gradle`：

- `minSdkVersion 17`
- 对应 Android 4.2 Jelly Bean

后续开发以 Android 4.2 / API 17 作为最低支持版本。实现约束如下：

- 不引入需要 AndroidX/AppCompat 的控件依赖。
- 不使用 API 17 以上的未保护调用；确需使用高版本 API 时必须通过 `Build.VERSION.SDK_INT` 判断。
- 使用 Java 8 语法时以当前 Gradle 配置为准，避免引入额外运行时库。
- U 盘与外部存储访问继续保留传统文件路径扫描，同时兼容模拟器 `/data/local/tmp/tvbooks` 测试目录。
- 面向本地安装和电视盒子使用，优先保证 Android 4.x 到 Android 9 的文件路径访问稳定。

## 3. U 盘目录规范

所有内容、配置、播放记录和收藏都必须位于 U 盘根目录的 `tvbooks` 文件夹内。应用可以读取和创建以下结构：

```text
tvbooks/
  menu.txt
  config/
    settings.properties
    playback.properties
    favorites.properties
  huiben/
    index.txt
    chitang/
      1.jpg
      1.mp3
      2.jpg
      2.mp3
  gushi/
    001.mp3
    002.mp3
```

目录约定：

- `menu.txt` 使用 UTF-8 JSON，定义一级菜单和二级菜单。
- `config/` 由应用自动创建，存放所有运行产生的数据。
- 绘本分类目录可以包含 `index.txt`，一行一个绘本目录名，用于固定播放顺序。
- 绘本页面使用同名序号文件，例如 `1.jpg` 和 `1.mp3`。页面可以只有图片或只有音频。
- 音频目录自动扫描非目录文件并按自然顺序播放。
- 应用不得把收藏、播放进度、用户设置作为主存储写入应用私有目录。旧版本 SharedPreferences 只允许作为迁移或 U 盘不可写时的临时兼容。

## 4. 菜单配置格式

`menu.txt` 示例：

```json
{
  "menu": [
    {
      "name": "huiben",
      "type": "audio_image"
    },
    {
      "name": "gushi",
      "type": "audio",
      "submenu": [
        { "name": "100shou" },
        { "name": "200shou" }
      ]
    }
  ]
}
```

字段说明：

- `name`：目录名，也是界面显示名。
- `type`：
  - `audio_image`：图片+音频绘本。
  - `audio`：纯音频目录。
- `submenu`：可选二级菜单。二级目录实际路径为 `一级目录/二级目录`。

应用会额外提供“我的收藏”入口，不要求用户在 `menu.txt` 中配置。

## 5. 遥控器操作规范

通用按键：

- `方向键`：移动焦点或切换内容。
- `OK/Enter`：主菜单中打开焦点项；播放器中暂停/继续。
- `Back`：返回上一页或退出播放器。
- `Menu`：播放器中显示/隐藏底部控制条。
- `Bookmark` / `*` / 红色功能键：收藏或取消收藏当前内容。

绘本播放器：

- `Left`：上一页。
- `Right`：下一页。
- `Up`：上一本。
- `Down`：下一本。
- `OK`：暂停或继续当前音频。
- `Menu`：显示/隐藏底部小控制条。
- `Bookmark` / `*` / 红色功能键：收藏/取消收藏当前绘本。

音频播放器：

- `Left` / `Up`：上一首。
- `Right` / `Down`：下一首。
- `OK`：暂停或继续。
- `Menu`：显示/隐藏控制条。
- `Bookmark` / `*` / 红色功能键：收藏/取消收藏当前音频。

界面按钮也必须支持同等操作，便于部分遥控器通过焦点按钮完成控制。

## 6. 收藏规范

收藏数据文件：

```text
tvbooks/config/favorites.properties
```

收藏项最少保存：

- 类型：`audio_image` 或 `audio`
- 相对路径：相对于 `tvbooks/` 的内容路径
- 标题：界面显示名称

行为要求：

- 绘本收藏到具体绘本目录，例如 `huiben/chitang`。
- 音频收藏到当前音频文件，例如 `gushi/100shou/001.mp3`。
- 再次按收藏键会取消收藏。
- 主菜单“我的收藏”根据收藏文件生成按钮列表。
- 打开收藏绘本时进入对应绘本，并尽量恢复上次页码。
- 打开收藏音频时进入对应音频目录，并定位到对应音频。

## 7. 断点续播规范

播放记录文件：

```text
tvbooks/config/playback.properties
```

绘本记录：

- 当前分类最后播放的绘本序号。
- 当前绘本最后播放的页码。
- 当前页音频播放进度，单位毫秒。

音频记录：

- 当前音频目录最后播放的文件序号。
- 当前音频文件播放进度，单位毫秒。

保存时机：

- 进入新绘本、新页面、新音频时保存索引。
- 应用暂停、返回、退出播放器前保存当前媒体进度。
- 手动切换下一页/下一首时，新内容的媒体进度重置为 0。

恢复策略：

- 从普通菜单进入分类时，恢复该分类最近播放位置。
- 从收藏进入时，优先定位收藏内容，再恢复该内容的页码或媒体进度。
- 如果记录指向的文件已不存在，自动回退到目录中第一个可播放内容。

## 8. 界面规范

主界面：

- 左侧为一级菜单和全局设置按钮。
- 中间为二级菜单或收藏列表。
- 右侧为操作说明和状态信息。
- 按钮高度、字号、间距需要适合电视远距离观看。
- 焦点按钮需要明显高亮。

播放器界面：

- 绘本图片应铺满可用区域并居中显示，默认状态下优先保证大屏沉浸感。
- 绘本播放器默认隐藏按钮，仅保留右下角轻量状态文字，显示书名、页码、收藏状态和播放模式。
- 按 `Menu` 键后，在播放页底部浮出一排小控制按钮；控制条不得占据主要阅读区域，再按 `Menu` 隐藏。
- 音频界面显示当前音频名、播放进度和收藏状态。
- 控制按钮使用遥控器可聚焦按钮，作为可见辅助入口；常用播放动作以遥控器方向键和 OK 键为主。

## 9. 实现原则

- 先保证离线 U 盘播放能力，再考虑在线 H5。
- 不破坏现有 `audio_image` 和 `audio` 菜单行为。
- 文件读写失败必须用 Toast 或状态文本提示，不应直接崩溃。
- 文件排序应稳定，数字文件名按自然数字顺序排列。
- 代码保持简单：优先使用 Java/Android 标准库，避免引入新依赖。
- 新增功能必须能在模拟器通过 `/data/local/tmp/tvbooks` 测试目录验证。

## 10. 验证清单

开发完成后至少验证：

- Gradle Debug 包可以成功构建。
- `minSdkVersion` 仍为 17。
- 模拟器存在 `/data/local/tmp/tvbooks/menu.txt` 时可读取测试菜单。
- 主菜单可用方向键移动焦点并打开内容。
- 绘本可以上下本、左右翻页、OK 暂停、Menu 显示/隐藏控制条、Bookmark 收藏。
- 音频可以上下/左右切歌、OK 暂停、Bookmark 收藏。
- 退出后再次进入可以恢复绘本页码和音频进度。
- 收藏文件、播放记录、设置文件写入 `tvbooks/config/`。
- U 盘目录不存在或菜单缺失时有明确提示，不崩溃。
