# TVBook 原型设计提示词

请为一个 Android TV 应用设计高保真交互原型，应用名称为 TVBook。它运行在安卓电视/电视盒子上，核心用途是从 U 盘根目录的 `tvbooks` 文件夹读取离线绘本、图片和音频，并通过遥控器进行播放、收藏和断点续播。

## 项目背景

TVBook 面向儿童家庭场景。家长把绘本图片、同名音频、音频故事和菜单配置预先放入 U 盘，插入电视后即可使用。应用主要运行在大屏电视上，用户距离屏幕较远，必须优先适配遥控器，而不是触摸屏。

最低系统版本需要兼容 Android 4.2 / API 17，因此原型应避免依赖现代手机式手势和复杂 Material 组件。视觉风格要简单、稳定、适合电视远距离观看。

## 核心功能

1. 主界面直接读取 `tvbooks` 下的目录，最多显示三级菜单和“我的收藏”，不依赖 `menu.txt`。
2. 内容类型包括：
   - `audio_image`：绘本，目录内按 `1.jpg + 1.mp3`、`2.jpg + 2.mp3` 方式播放。
   - `audio`：纯音频目录，自动播放音频列表。
3. 支持收藏：
   - 绘本收藏到具体绘本目录。
   - 音频收藏到具体音频文件。
   - 收藏入口叫“我的收藏”。
4. 支持断点续播：
   - 绘本恢复到上次书本、页码和音频进度。
   - 音频恢复到上次曲目和播放进度。
5. 所有用户数据都保存在 U 盘 `tvbooks/config/` 下，包括设置、收藏和播放记录。

## 遥控器交互

主界面：

- 方向键移动焦点。
- OK/Enter 打开焦点项。
- Back 退出或返回。

绘本播放页：

- 默认是沉浸式阅读，不显示大面积按钮。
- Left：上一页。
- Right：下一页。
- Up：上一本。
- Down：下一本。
- OK/Enter：暂停或继续当前音频。
- Menu：显示或隐藏底部控制条。
- Bookmark / 星号 / 红色功能键：收藏或取消收藏当前绘本。

音频播放页：

- Left 或 Up：上一首。
- Right 或 Down：下一首。
- OK/Enter：暂停或继续。
- Menu：显示或隐藏控制条。
- Bookmark / 星号 / 红色功能键：收藏或取消收藏当前音频。

## 原型页面要求

请至少设计以下页面：

1. 主界面
   - 左侧为主菜单列表，包含“我的收藏”和根目录中的分类。
   - 保持固定三栏；二级和三级菜单在中间区域切换显示。
   - 右侧为状态信息，例如 Android 版本、U 盘路径、播放设置。
   - 当前焦点必须明显，高亮状态适合遥控器操作。
   - 不要做营销落地页，第一屏就是可用的电视应用界面。

2. 绘本播放页（默认沉浸状态）
   - 图片尽可能大，保持比例，居中显示。
   - 背景为黑色或深色，减少干扰。
   - 只保留右下角轻量状态文字，例如“池塘 第 2 页 已收藏 自动”。
   - 不显示大按钮，不遮挡主要画面。

3. 绘本播放页（Menu 打开控制条）
   - 底部浮出一排小控制按钮。
   - 控制条半透明、低高度，不超过屏幕高度的 10%。
   - 按钮包括：上一页、下一页、上一本、下一本、暂停/继续、收藏、自动/手动。
   - 控制条不应让用户感觉页面从沉浸阅读变成工具界面。

4. 音频播放页
   - 显示当前音频标题、播放进度、收藏状态。
   - 视觉比绘本页更简洁，可以有轻量背景图形，但不要喧宾夺主。

5. 缺少 U 盘或目录为空的提示状态
   - 文案要短，适合电视用户理解。
   - 提供清晰的“重新检测”或“退出”入口。

## 视觉设计要求

- 设计比例优先按 16:9 横屏电视，至少覆盖 1920x1080。
- 字体要大，远距离可读；主菜单文字建议 30sp 以上，播放页状态文字可更小。
- 焦点高亮必须非常明确，但不要花哨。
- 播放器页重点是内容沉浸感，按钮和状态都应克制。
- 不使用手机底部导航、抽屉菜单、触摸手势作为主交互。
- 不使用复杂渐变、装饰光斑、过度卡片化布局。
- 色彩建议深色背景、清晰焦点色、温和高对比。
- 所有文字要能在中文环境下完整显示，不要溢出或被裁切。

## 输出格式

请输出可交互原型设计方案，包含：

- 页面结构说明。
- 每个页面的布局细节。
- 遥控器焦点流转说明。
- 播放页 Menu 控制条的显示/隐藏交互。
- 关键状态：未收藏/已收藏、自动/手动、暂停/播放、缺少 U 盘。
- 如能生成 HTML/CSS/JS 原型，请使用纯前端静态文件实现，不依赖外网资源。
